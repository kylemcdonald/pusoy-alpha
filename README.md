# Pusoy Alpha

Pusoy Alpha is a two-player Pusoy Dos implementation with a browser opponent driven by Monte Carlo tree search and a small policy/value network trained from AlphaZero-style self play.

The playable app is published with GitHub Pages at `https://kylemcdonald.github.io/pusoy-alpha/`.

## What Is Included

- A heads-up Pusoy Dos rules engine in TypeScript.
- A handcrafted baseline policy/value model.
- An MCTS search implementation that can use either the baseline or neural model.
- A browser neural inference path for `public/models/neural-policy.json`.
- A CUDA PyTorch AlphaZero-style trainer in `scripts/gpu_train_alphazero.py`.

The previous tabular learner, writeup page, and self-play web UI have been removed.

## Baseline Strategy

The baseline is implemented by `HandcraftedModel` in `src/ai/model.ts`. It assigns every legal move a heuristic prior, normalizes those scores, and estimates value from hand count, hand strength, and lead control.

Move priorities:

- Passing gets a small fixed score of `0.08`.
- Invalid fallback plays score `0.01`.
- Legal plays start at `0.25 + 0.18 * cardsPlayed`, so larger combinations are preferred over smaller ones.
- A move that empties the hand gets `+5.0`.
- A move that leaves two or fewer cards gets `+0.8`.
- When following an active table combination, lower winning combinations get up to `+0.45`.
- When leading a trick, lower combinations get up to `+0.7`.
- Spending a `2` before going out is penalized by `-0.2`.
- If the next player has one card and the candidate move is a single, stronger singles get an extra blocking bonus of `comboStrength * 0.6`.

Value priorities:

- Terminal states use the actual game reward: `+1` for first place and `-1` otherwise.
- Nonterminal value starts from card-count advantage: `(averageOpponentCards - myCards) / 13 * 1.25`.
- It adds average hand strength, normalized by card power, with weight `0.22`.
- It adds `+0.12` when the player to evaluate has the lead and there is no active table combination.
- The final value is clamped to `[-1, 1]`.

## AlphaZero-Style Learner

The neural learner uses games generated by MCTS. At each decision, MCTS expands legal moves using the neural policy prior and value estimate. The policy head is trained to match normalized root visit counts, and the value head is trained on the terminal game outcome for the acting player.

The shipped checkpoint is `public/models/neural-policy.json`:

- Name: `gpu-alphazero-rollout-hybrid`
- Created: `2026-05-31T09:00:07Z`
- Device: CUDA
- Training style: `alphazero-mcts-visit-targets`
- Requested training time: 30 minutes
- Actual elapsed time: 1803.525 seconds
- Training games: 17,984 against a searched handcrafted opponent
- Training transitions: 175,819
- MCTS simulations per move: 24
- Handcrafted opponent search simulations: 12
- Replay size: 80,000 transitions
- Train samples per iteration: 4,096
- Learning rate: `0.00001`
- Inference blend: handcrafted prior weight `0.6`, neural policy temperature `0.5`, greedy rollout value weight `0.25`
- Arena result: 60.1% first-place rate over 1,000 games against the handcrafted baseline with 48 simulations per move and seed `20260531`

This checkpoint is not a solved or proven-optimal Pusoy Dos strategy. It is a trained policy/value approximation used as a search prior and value estimator.

## Neural Encoding

The active model uses a 216-value state vector:

- 52 bits for the current player's hand.
- 52 bits for cards previously played by the current player.
- 52 bits for cards previously played by the opponent.
- 52 bits for the active table combination.
- 6 scalar state features: own hand size, opponent hand size, active combo size, lead flag, passes since last play, and capped turn count.
- 2 active-owner flags: active combo was played by self, active combo was played by opponent.

Each candidate move uses a 62-value vector:

- 52 bits for cards in the move.
- 1 pass flag.
- 1 normalized move-size scalar.
- 8 one-hot combination kind values: single, pair, triple, straight, flush, full house, four of a kind, straight flush.

## Development

Install dependencies:

```bash
npm ci
```

Run the local app:

```bash
npm run dev -- --host 0.0.0.0 --port 8080
```

On the development machine, open `http://vibecheck.local:8080/`.

Run tests and build:

```bash
npm test
npm run build
```

Train a new AlphaZero-style checkpoint:

```bash
npm run train:alphazero -- --duration-minutes 60 --batch-games 64 --simulations 24 --train-samples 4096 --minibatch-size 256 --lr 0.00005 --resume
```

The trainer writes browser-loadable JSON weights to `public/models/neural-policy.json` by default.
